# 3.1 認識飼料等級：你的特務大腦由誰決定？ 🦞

在 OpenClaw 的世界裡，**模型** 就是你的數位龍蝦所吃的「飼料」。

就像現實中的養殖場一樣，高品質的飼料（先進的 LLM 模型）能讓龍蝦生長得壯碩、反應靈敏且具備高階智慧；而廉價甚至不合格的飼料，則會讓你的特務在執行複雜任務時顯得「笨笨的」，出現邏輯斷層或無法理解隱含意圖。

### 數位飼料清單：主流模型點點名

要養出一隻頂級特務，你得先認識目前市場上主流的四種飼料等級：

| 飼料品牌                             | 特點                 | 適用場景                   |
| :------------------------------- | :----------------- | :--------------------- |
| **GPT（頂級海味）**                    | 綜合力最強，邏輯極其嚴密。      | 複雜商務決策、程式開發、跨學科分析。     |
| **Claude （文青特餐）**                | 文風優雅、邏輯平滑，長文理解力極佳。 | 內容創作、腳本撰寫、長篇文件整理。      |
| **Gemini（全能大餐）**                 | 原生多模態支援，支援超長的上下文。  | 音訊/視訊分析、一次丟進整本書的檢索引導。  |
| **MiniMax / DeepSeek（高 CP 值配方）** | 中文語境理解極強，成本低廉。     | 一般日常聊天、簡單摘要、通用的中文特務任務。 |

### 養殖者的第一個難題：為什麼不全部給最好的？

你可能會想：「既然 GPT-4o 這麼強，我全部都用頂級飼料不就好了？」

**答案是：成本與浪費。**

如果你只是要你的龍蝦幫你「查詢明天的天氣」或是「把這封信改成比較禮貌的語氣」，這種雜魚級的任務，用 GPT-4o 就像是用波士頓龍蝦去餵養一般池塘小蝦，雖然有效，但你的荷包會很快乾涸。

### 飼料與智力的關係

實務上，如果你發現你的特務在執行任務時一直轉圈圈，或者給出的答案答非所問，通常是因為你給的「飼料智商」不足以支撐這項任務。

*   **初級任務**：處理單一指令（改錯字、查表、翻譯短句）。
*   **中級任務**：需要多步推理（摘要長文、模擬對話、規劃行程）。
*   **高級任務**：需要理解上下文隱喻、複雜邏輯與工具調用。

**「便宜飼料讓龍蝦笨笨的」** 是一條養殖鐵律。當你發現特務失靈時，第一個要檢查的就是你的 `openclaw.json` 裡面，是不是給牠分配了過於弱小的模型種子。

你應該檢查設定檔中的 `model` 欄位：

```json
{
  "agents": {
    "defaults": {
      "model": {
        "primary": "openai/gpt-4o" // 這是全域預設大腦
      }
    },
    "list": [
      {
        "id": "market-analysis",
        "model": "anthropic/claude-3-5-sonnet-latest" // 特定特務的大腦覆蓋
      }
    ]
  }
}
```

---

{% hint style="info" %}
**營養師筆記**
在 OpenClaw 設定中，你可以為不同的 agent 分別設定不同的模型。讓主控 agent 吃好一點，讓負責雜事的子 agent 吃普通點，這就是專業養殖者的「營養配方」。
{% endhint %}
